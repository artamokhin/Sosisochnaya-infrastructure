Концепция возможной реализации приложения Сосисочной с использованием нескольких облачных провадеров.

Плюсы размещения компонентов приложения в облаках от разных провайдеров:

1. Снижение расходов на обслуживание за счет выбора оптимального провайдера (тарифа) под одну задачу.
2. Снижение времени отклика и повышение доступности для клиентов из определенного региона за счет запуска копий компонента приложения в разных регионах (в разных облаках).
3. Обеспечение выполнения региональных законодательных ограничений только для некоторых компонентов приложения.
4. Повышение времени доступности приложения при дублировании компонентов приложения в разных облаках.

Минусы такого решенния:

1. Сложнее администрировать и контролировать работу прилоения на нескольких облаках с различными интерфейсами и API
2. Растут задержки сетевого взаимодействия между компонентами приложения
